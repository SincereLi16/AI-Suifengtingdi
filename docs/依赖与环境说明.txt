四、OCR（`battle_ui_ocr.py` — PaddleOCR）
--------------------------------------------------------------------------------
  代码依赖：`from paddleocr import PaddleOCR`

  何为 OCR：光学字符识别（Optical Character Recognition），把截图里「文字/数字」像素区域
  还原成字符串。本项目的对局 UI 脚本在固定 ROI 上裁剪后调用 OCR，用于读阶段、等级、金币等。

  与模板匹配的区别：模板匹配是「找一张小图在整张图里的位置」；OCR 是「读这块区域里写了什么字」。
  对局 HUD 文本多、数字多，用 OCR 更合适。

  包名（pip 安装名）           版本（当前环境）     说明
  -------------------------   ------------------   --------------------------------
  paddleocr                   （视安装而定）       需单独安装；另需安装 PaddlePaddle 本体
  paddlepaddle                （视安装而定）       CPU 版即可满足截图 ROI 识别

  推荐组合（固定 UI、中文 + 数字、Windows CPU，优先稳定、少踩坑）：
  • 使用 PaddleOCR 2.x 经典管线（`ocr.ocr(img, cls=True)`），避免 3.x 默认走 PaddleX + 多模型
    与 `predict`/oneDNN 等环境差异。
  • 使用项目内专用虚拟环境，避免与全局 protobuf、torch、Google API 等冲突。

  推荐运行方式（无需手动 activate；其它脚本仍用你平时的全局 Python）：
  • 直接：python battle_ui_ocr.py  [参数见脚本 --help]
  • 脚本会在启动时检测：若当前解释器不是 .venv_battle_ocr，则自动创建该 venv（仅首次）、安装
    paddlepaddle、paddleocr、opencv-python、numpy、pillow，并用该 venv 重新启动自身。
  • 若要在当前全局解释器内直接跑（不经过 venv），设置环境变量：BATTLE_UI_OCR_NO_VENV=1，
    并自行保证已安装兼容的 paddle / paddleocr 等且无与其它包的冲突。

  示例（CPU；若默认 PyPI 无对应 Windows wheel，请按 Paddle 官网「安装」页选择系统与 Python 版本后复制命令）：
    pip install "paddlepaddle==2.6.2"
    pip install "paddleocr>=2.6.1,<3"

  若已安装 PaddleOCR 3.x：可卸载后按上式重装 2.x，或保留 3.x 并依赖 `battle_ui_ocr.py` 内对
  `predict`/`FLAGS_use_mkldnn` 的兼容处理（脚本已做兼容，但 2.x 更省事）。

  说明：仅改 paddlepaddle / paddleocr 版本时，一般不动 numpy、opencv、torch、requests 等其它包。
