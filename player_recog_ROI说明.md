# player_recog 主图 ROI 清单

本文档依据 `player_recog.py` 当前实现整理，坐标系为主图像素坐标，矩形格式统一为 `(x1, y1, x2, y2)`（左上到右下，右下为开区间上限）。

## 1) 固定中心 ROI（CenterRoi）

这些区域在 `_default_layout()` 中定义，中心点+半宽高固定：

| 字段 key | 名称 | 中心(cx,cy) | 半宽/半高(half_w,half_h) | 实际尺寸(宽x高) | 换算矩形 |
|---|---|---:|---:|---:|---|
| `phase` | 阶段 | (808, 30) | (52, 28) | 104x56 | (756, 2, 860, 58) |
| `level` | 等级 | (423, 1000) | (38, 36) | 76x72 | (385, 964, 461, 1036) |
| `exp` | 经验 | (540, 1000) | (54, 36) | 108x72 | (486, 964, 594, 1036) |
| `gold` | 金币 | (1150, 1000) | (52, 32) | 104x64 | (1098, 968, 1202, 1032) |
| `streak` | 连胜/连败 | (1256, 1000) | (50, 32) | 100x64 | (1206, 968, 1306, 1032) |

说明：
- 上表矩形按参考主图分辨率直接计算；实际运行会被 `to_rect()` 裁剪到图像边界内。
- `gold` 与 `streak` 在 OCR 阶段还会做内部二次裁剪/增强（不是额外主 ROI）。

## 2) 固定矩形 ROI（RectRoi）

同样在 `_default_layout()` 中定义：

| 字段 key | 名称 | 固定矩形(x1,y1,x2,y2) | 尺寸(宽x高) | 网格参数 |
|---|---|---|---:|---|
| `bonds` | 羁绊栏 | (110, 100, 470, 700) | 360x600 | `grid_cols=1, grid_rows=7` |
| `hp_nick` | 血量/昵称 | (1910, 100, 2190, 880) | 280x780 | `grid_cols=0, grid_rows=0` |

## 3) 羁绊栏动态 ROI（bonds）

`bonds` 支持动态候选选框（文本密度打分）：

- 候选 1：`(260, 110, 480, 690)`
- 候选 2：`(180, 110, 380, 690)`

并带有副图偏移逻辑：
- 若文件名 stem 为 `*-b` 或 `*_b`，整体 `y` 下移 `+10px`（`BOND_ROI_AUXILIARY_DY_PX = 10`）。

当前代码还支持固定模式（关闭动态候选）：
- 使用 `RectRoi` 基准框 `(110,100,470,700)`，同样会叠加上述副图 `dy`。

## 4) 血量/昵称内部子 ROI（hp_nick）

`hp_nick` 识别时并不直接在整块里一次性读完，而是拆成子 ROI：

### 4.1 血量列覆盖框（按分辨率缩放）

- 参考框：`(1970, 110, 2140, 880)`
- 通过 `_scale_hp_rect_ref_to_image()` 按当前图尺寸从参考分辨率缩放：
  - 参考分辨率：`2196 x 1253`

### 4.2 Top1~Top8 固定血量数字框（每行一个）

- 每行数字框参考尺寸固定 `60x36`
- 中心 `x = 2075`
- 行中心 `y` 列表：`[156, 277, 371, 467, 560, 654, 748, 844]`
- 这 8 个框都按图像实际分辨率缩放后使用

### 4.3 昵称 ID 子框

- 对每一行，ID 框宽高固定：`300x40`
- `ix1 = fx1 - 120`（再夹紧到图像范围）
- `iy2` 根据该行血量数字检测框顶部动态确定（`hpy_abs - 2`）
- `iy1 = iy2 - 40`

### 4.4 “我”行扩展血量框（兜底）

当某行 ID 为空并归一为“我”时，血量数字框进行二次扩展重识别：
- 向左扩展：`40px`
- 向上扩展：`40px`

## 5) 快速复用建议（给拼图方案）

若要做“裁块拼图一次 OCR”，可优先选这些稳定块：

- 直接使用第 1 节 5 个 `CenterRoi` 转矩形
- 使用 `bonds` 的最终选中矩形（动态或固定）再切 1x7
- 使用 `hp_nick` 的 8 个固定血量数字框 + 每行 ID 框

这样能保持与当前解析逻辑一致，便于后续做 A/B 对比。
